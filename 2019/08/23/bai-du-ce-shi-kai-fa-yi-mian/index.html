  11. 计算机网络的层次结构
    网络需要解决的问题:
    (1) 保证数据通路顺畅
    (2) 识别目的计算机
    (3) 目的计算机状态
    (4) 数据是否错误
    分层实现不同的功能
    基本原则:
    各层之间是相互独立度
    每一层要有足够的灵活性
    各层之间完全解耦
    OSI 七层模型
    应用层 为计算机用户提供接口和服务
    表示层 数据处理(编码解码、加密解密)
    会话层 管理(建立、维护、重连) 通信会话
    传输层 管理端到端的通信
    网络层 数据路由
    数据链路层 管理相邻节点的数据通信
    物理层 数据通信的光电物理通信
    TCP / IP 四层模型
    应用层 HTTP / FTP / SMTP / POP3
    传输层 TCP/UDP
    网络层 IP/ICMP
    网络接口层 Ethernet / PPP
    TCP / UDP
    传输层 : 进程与进程的通信
    使用端口来标记不同的网络进程 Port 16比特位(0~65535)
    UDP协议详解 UDP协议详解
    对数据不合并 不拆分
    UDP 首部
    16位源端口号:
    16位目的端口号:
    16位UDP 长度:
    16位UDP 校验和
    UDP 数据

UDP是无连接协议:直接发送数据 无需连接
UDP 不能保证可靠的交付数据
UDP 是面向报文传输的 :直接把应用层数据塞进UDP数据报的数据
UDP 没有拥塞控制 不管网络是否拥挤
UDP 首部开销比较小 8个字节

TCP协议 传输控制协议
TCP是面向连接的协议
TCP的一个连接有两端(点对点通信)
TCP提供可靠的传输服务
TCP协议提供全双工的通信
TCP是面向字节流的协议:以字节位单位处理数据
TCP协议详解
头部固定有20个字节
16位源端口 16位目的端口
序号 02^32-1 数据报第一个字节的序号
确认号 02^32-1 期待收到的首字节序号
确认号为N : 则表示N-1序号的数据都已收到

数据偏移 占4位 015 单位为32位字 数据偏离首部的距离
15 *4 = 60
2060个字节
TCP标记 占6位
URG\ACK\PSH\RSY\SYN\FIN
URG: 紧急位
ACK: 确认位 ACK = 1 确认号生效
PSH : PSH = 1 尽快的把数据传送到应用层
RST: RST = 1: 重新建立连接
SYN: SYN=1 表示连接请求报文
FIN: FIN=1 表示释放连接
窗口
占16位 02^16-1
窗口指明对方允许发送的数据量
确认号 窗口长度
501 100 5011500
紧急指针 紧急数据在报文的位置 URG=1
TCP选项 最多有40个字节 支持未来的拓展

可靠传输的基本原理

  1. 停止等待ARQ协议
    超时重传，接受确认消息异常
    发送的消息丢失、确认消息丢失、确认消息很久才到
    可靠传输
    超时定时器: 每发送一个消息，都需要设置一个定时器
    停止等待协议对信道的利用率不高
  2. 连续ARQ协议(自动重传请求)
    滑动窗口 累计确认

TCP可靠传输基于连续ARQ协议
TCP滑动窗口以字节位单位
选择重传: 指定重传的字节的边界

TCP协议的流量控制
让发送方发送的速率不要太快
流量控制是使用滑动窗口来实现 的
