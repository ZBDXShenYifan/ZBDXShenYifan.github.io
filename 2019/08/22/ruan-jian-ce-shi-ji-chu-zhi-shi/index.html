软件的概念

软件是计算机系统中与硬件相互依存的另一部分，它是包括程序，数据及其相关文档的完整集合
程序:
数据
文档

软件的十大特性

形态特性、智能特性、开发特性、
质量特性:存在缺陷
生产特性:复制简单 成本低
管理特性:
环境特性: 软件开发和运行需要计算机系统的环境
维护特性: 升级、优化、功能更新、重构
废弃特性: 不会因被“用坏”而被废弃
应用特性: 应用极为广泛

软件的分类

系统软件: 操作系统、数据库管理软件
应用软件: QQ 图像浏览器

软件的生命周期

问题定义->可行性研究->需求分析->概要设计->详细设计->编码和单元测试->综合测试->软件维护

软件开发模型

瀑布模型

计划->需求分析->设计->编码->测试->运行维护
特点 :

  1. 软件开发的各项活动严格按照线性方式进行。
  2. 当前活动接受上一项的工作结果 。
  3. 当前活动的工作结果需要进行验证

缺陷:

  1. 增加开发的风险
  2. 早期的错误可能要等到开发后期的阶段才能发现

原型模型

开发周期长。开发会收到需求变更的影响
特点:

  1. 实现客户与系统的交互
  2. 进一步细化软件开发需求
  3. 开发人员可以确定客户的真正需求是什么

螺旋模型

制定计划 -> 风险分析 -> 实施工程 -> 客户评估
特点:

  1. 螺旋模型是将瀑布模型与快速原型模型结合起来
  2. 强调了其他模型所忽视的风险分析
  3. 每一次螺旋包括4个步骤: 制定计划 -> 风险分析 -> 实施工程 -> 客户评估

缺点:
风险用户不太能接受

敏捷模型

特点: 短周期开发 增量开发
由程序员和测试人员编写的自动化测试来监控开发进度
通过口头沟通、测试先行
人员素质要求比较高
测试员掌握各种脚本语言脚本编程、能执行单元测试、自动化测试

软件测试

在规定条件下对程序进行操作 已发现程序错误 衡量软件质量 评估
弄清预期结果与实际结果的区别

目的

发现问题，检查系统是否满足需求

软件测试的方法和分类

按生命周期划分

单元测试: 编码人员测试
冒烟测试: 主要流程测试
集成测试:
系统测试:
验收测试
按测试方法划分:
白盒测试:静态测试、动态分析: 逻辑覆盖测试、插桩测试
黑盒测试: 功能测试、自动化测试、性能测试、安全测试
灰盒测试:
其他:随机测试(客户、非专业人员测试)、探索性测试、阿尔法测试 β测试(验收类型测试)

软件测试属于

C/S 客户端软件/服务端软件
B/S 浏览器/服务器
测试环境 : 软件 + 硬件 + 网络
测试用例: 输入(数据和操作步骤) + 输出(预期结果) + 测试环境
冒烟测试: 在大规模测试前先进行基本的测试，看是否具有可测性
α测试: 内部测试
β测试: 最终用户测试 公测

软件测试模型

V模型 开发测试并行进行，编码之后
W模型 测试过程伴随整个开发周期，测试开发同步执行
H模型 测试级别之间不存在次序关系
X模型 分块化探索性测试，分成程序片段

软件测试覆盖率

覆盖率 = (至少被执行一次的item数) / item总数

  1. 通过覆盖率数据，可以检测我们的测试是否充分
  2. 分析出测试的弱点在哪里
  3. 指导增加测试用例

需求覆盖: 需求点被验证
用例覆盖: 验证通过的用例数量 / 总的用例总数 80%

软件测试的原则

  1. 所有的测试都应该追溯到用户的需求
  2. 尽早启动测试工作
  3. Pareto法则
  4. 穷尽测试是不可能的
  5. 杀虫剂怪事
  6. 前进两步，后退一步
  7. 三心二意 细心、信心、耐心 沟通意识、缺陷预防意识

软件测试规范

  1. 测试管理 资源管理 配置管理 测试规划 测试实施
  2. 角色的确定、进入的准则、输入项、活动过程、输出项、验证与确认、退出的准则
